설명

Dijkstra 는 시작점으로부터 나머지 모든 정점에 대한 거리를 구하는 알고리즘이다.

단 간선의 가중치가 음수가 아니어야 한다. 만약 음수를 포함한다면 무한루프를 돌게 된다.

알고리즘은 대락 다음과 같다.

  1. 일차원 정수 배열인 dp[] 를 두어 전부 무한대 값을 할당한다.
  2. 출발 정점 인덱스를 s 라고 할 때 dp[s] = 0 으로 할당한다. 그리고 s 와 인접한 정점을 컨테이너에 담는다.
  3. 컨테이너 안의 정점이 없을 때 까지 루프를 돈다.
    1. 가장 작은 dp[u] 를 갖는 컨테이너 안의 정점 u 를 꺼낸다.
    2. u 와 인접한 정점 v 를 잇는 간선의 최소가중치가 edge(u,v) 라고 하자. dp[v] > dp[u] + edge(u,v) 라면
      1. dp[v] = dp[u] + edge(u,v) 를 수행한다.
      2. v 를 컨테이너에 넣는다.
  4. dp[] 의 값은 방문 가능한 경우 최단거리 그렇지 않은 경우 무한대가 된다.

아이디어는 Gridy + DP 라고 생각할 수도 있다.

위의 절차의 1.3.1 단계에서 가장 작은 값을 어떻게 찾는 지가 중요하다. 그 방법에 따라 위와 같이 시간복잡도가 달라진다.

증명

Loop Invariant

임의의 방문한 정점인 $$v$$ 에 대하여 $$\text{dp}[v]$$ 는 $$v$$ 까지의 최단거리이고,
임의의 방문하지 않은 정점인 $$u$$ 에 대하여 $$\text{dp}[u]$$ 는 방문한 점만을 거쳤을 때의 $$u$$ 까지의 최단거리이다.

이때 방문한다는 말의 의미는

  1. 시작점이거나
  2. 위 알고리즘에 의해 해당 정점이 가진 간선에 대하여 계산을 한 경우

를 말한다.

Base case

시작점 에 대해 는 초기 세팅으로 0 이 된다. 자기 자신과의 거리는 0 이므로 이 값은 최단거리가 자명하다.

Inductive Step

개의 정점에 대하여 탐색이 완료되었을 때 에서도 위 알고리즘이 Loop Invariant 를 만족시킴을 보이면 된다. 이는 크게 3단계로 나눌 수 있다.

  • 를 이번에 탐색할 정점이라고 하자. 값이 최단거리임을 보이자
    • 귀류법을 사용하여 가 최단거리가 아니라고 하자. Loop Invariant 상 이 값은 개의 정점들만 사용한 경로의 최단거리이다. 이 경로가 전체 정점을 고려한 최단경로가 아니라면 탐색된 개의 정점 외의 다른 정점을 지나야한다. 이 중 최초로 방문하지 않은 정점을 라고 하자. 그러면 라고 할 수 있다. 는 음이 아니므로 가 성립한다. 그러면 모순이 발생하는데 가 더 크다면 가 이번에 탐색될 정점이기 때문이다. 증명완료.
  • 와 인접한 정점 에 대하여 가 갱신이 된다. 가 를 포함한 방문한 정점들만 이용한 최단거리임을 보이자.
    • 방문한 정점만을 사용한 까지의 최단경로가 를 포함하지 않는다고 해보자. 그럼 갱신하기 전부터 이미 방문한 정점만을 사용한 최단거리이다. 반대로 포함한다고 해보자. 그러면 기존 값보다 더 짧아졌으니 더이상 짧을 수가 없다. 증명완료.
  • 나머지 값들은 변하지도 영향을 받지도 않으므로 생략.
